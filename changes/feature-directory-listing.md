# Feature: Directory Listing (autoindex)

Overview

Add directory listing (autoindex) support so that when a request targets a directory with no index file, the server returns an HTML listing of files and subdirectories.

Scope

- Serve an autogenerated HTML index when `public/<path>/` exists and no `index.html`/`index.htm` present.
- Include links to files and directories (directories should include trailing slash in links).
- Ensure Content-Type is `text/html; charset=utf-8` and proper `Content-Length` is set.
- Protect against path traversal (reuse existing checks).
- Keep listings simple and lightweight (no templating in v1).

Files to modify

- `src/main.zig` — update `serveFile` to call new `serveDirectoryListing` when `tryIndexFile` returns null.
- `src/http/response.zig` — no change required unless helper becomes necessary (use current response builder for HTML responses).
- `test/` or inline tests in existing modules — add integration + unit tests for directory listing behavior.

Design notes

- Implementation will build the listing into an in-memory buffer (ArrayList) and then send it with a Content-Length header. For very large directories this can be optimized later to stream with chunked encoding.
- Links will be URL-escaped minimally for common characters.

Testing plan

1. Unit test: call new `serveDirectoryListing` with a small test directory and assert the response contains expected filenames and `Content-Type: text/html`.
2. Integration test: create `public/test-listing/` with sample files, request it and verify returned HTML includes links.

Acceptance criteria

- Requesting a directory with no index returns 200 and an HTML body containing links to entries.
- Directories are linked with trailing slash and regular files without.
- No regression in existing file-serving tests.
